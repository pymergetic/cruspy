# cruspy

**cruspy ‚Äî polyglot shared-memory runtime**

C++ data layer, Rust mediation bridge, Python/Pydantic on top. Define models in
``types.hpp`` (``CRUSPY_MODEL``); **easybind** codegen fills Rust and Python (EP-0012).

| Surface | Value |
|---------|-------|
| **Repository** | [github.com/pymergetic/cruspy](https://github.com/pymergetic/cruspy) |
| **PyPI** | ``pymergetic-cruspy`` (when published) |
| **Import** | ``pymergetic.cruspy`` |
| **Monorepo path** | ``packages/cruspy/`` |

## Architecture (Option A)

| Concern | Owner |
|---------|--------|
| Parse ``CRUSPY_MODEL``, emit ``mod.rs`` / ``__init__.pyi`` / ``errors.rs`` | **easybind** (`easybind-rust-module`) |
| 3-layer runtime (cxx + PyO3 + Pydantic, SHM, phase gating) | **cruspy** |
| 2-layer C++‚ÜîPython (nanobind) | **easybind** (cppdantic, etc.) |

**cruspy does not link easybind or nanobind at runtime.** easybind is a **build-time**
codegen tool only.

## Status

**Phase 1 (monolith, heap default)** ‚Äî codegen-driven ``Document`` and ``Token`` models:

- Edit ``src/pymergetic/cruspy/models/*/types.hpp`` ‚Üí run codegen ‚Üí rebuild
- C++ validation in ``mod.cpp``, Rust bridge + Pydantic from ``generated/``
- Error hierarchy mirrored from ``errors/mod.hpp`` via codegen

## Quick start (dev)

```bash
cd packages/cruspy
uv sync
bash scripts/codegen.sh          # or: build.rs runs this automatically
uv run --with maturin maturin develop --release
uv run --with pytest pytest -v
python -c "from pymergetic.cruspy.models.document import Document; print(Document(id=1, text='hi', score=0.5))"
```

### Codegen

```bash
# Regenerate bindings from C++ CRUSPY_MODEL sources
bash scripts/codegen.sh

# CI staleness check (generated/ must match types.hpp)
bash scripts/check_codegen.sh
```

Requires ``pymergetic-easybind`` in the active environment (monorepo ``uv sync`` provides it).
